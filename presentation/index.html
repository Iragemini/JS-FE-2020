There are only three types of conversion in JavaScript:

  • to string
  • to boolean
  • to number

JavaScript variables can be converted to a new variable and another data type:

  • By the use of a JavaScript function (Explicit coercion)
  • Automatically by JavaScript itself (Implicit coercion)

String()

The String() function converts the value of an object to a string.
Note: The String() function returns the same value as toString() of the individual objects.

Syntax: String(object)

Code examples: 

									
										var number = 12;

										var newString = String(number);	// returns “12”;

toString()

The toString() method converts a number to a string.

Syntax: number.toString(radix)

radix: optional. Which base to use for representing a numeric value. Must be an integer between 2 and 36.
2 - The number will show as a binary value
8 - The number will show as an octal value
16 - The number will show as an hexadecimal value

Code examples: 

									
										var number = 12;

										number.toString(2);	// returns “1100”
 
										number.toString(8);	// returns “14”
										
										number.toString(16);	// returns “c”

For null and undefined values returns TypeError

toFixed ()

For number to string conversion only

The toFixed() method converts a number into a string, rounding to a specified number of decimals.
Note: if the desired number of decimals are higher than the actual number, zeros are added to create the desired decimal length.

Syntax: number.toFixed(x)

x: optional. The number of digits after the decimal point. Default is 0 (no digits after the decimal point).

Code examples: 

									
										var number = 123456.789;

										number.toFixed(3);	// returns “123456.789”

toExponential ()

For number to string conversion only

The toExponential() method converts a number into an exponential notation.

Syntax: number.toExponential(x)

x: optional. An integer between 0 and 20 representing the number of digits in the notation after the decimal point. If omitted, it is set to as many digits as necessary to represent the value.

Code examples: 

									
										var number = 123456.789;

										number.toExponential(1);	// returns “1.235e+5”

toPrecision ()

For number to string conversion only

The toPrecision() method formats a number to a specified length.

Syntax: number.toPrecision(x)

x: optional. The number of digits. If omitted, it returns the entire number (without any formatting).

Code examples: 

									
										var number = 123456.789;
	
										number.toPrecision(7);	// returns “123456.8”

Implicit conversions

  • Implicit coercion is triggered by the binary + operator, when any operand is a string. 

    											
												var boolValue = true;
			
												console.log(boolValue + " story");// returns “true story”

  • Template strings bring simple string interpolation to JavaScript. That is, they’re a nice-looking, convenient way to plug JavaScript values into a string. 
    										
											var val;
		
											console.log(`value is ${val}`);// returns “value is undefined”
	
											var val = null;
		
											console.log(`value is ${val}`);// returns “value is null”
	
											var val = 2021;
		
											console.log(`Happy new ${val} year!`);// returns “Happy new 2021 year!”

Symbols to String

  • Symbols can converted to string only explicity 
    										
											var symbol = Symbol.for("mySymbol");
											
											symbol.toString();	// returns 'Symbol(mySymbol)'
											
											String(symbol);	// returns 'Symbol(mySymbol)'
  • If you try to concatenate the symbol directly with a string, an error will be thrown 
    										
											var symbol = Symbol.for("mySymbol");
											
											symbol + " ";	// returns TypeError

to Boolean conversion

The abstract operation ToBoolean (argument) takes argument argument. It converts argument to a value of type Boolean according to Table 2

Table 2: ToBoolean Conversions

There are two ways to convert values to Boolean in JavaScript:

  • Boolean() function
  • Implicit conversion (|| && !)

Boolean ()

Boolean () function returns the boolean value of variable. It can also be used to find boolean result of a condition, expression etc.

Syntax: Boolean(variable/expression)

Note:A variable or object which has value are treated as true boolean values. ‘0’, ‘NaN’, empty string, ‘undefined’, ‘null’ are treated as false boolean values.

Code examples: 

									
										Boolean(10);	// returns true
	
										Boolean('false');	// returns true

										Boolean('');	// returns false

										Boolean(1 < 2);	// returns true

Implicit conversion

Implicit conversion happens in logical context, or is triggered by logical operators ( || && !) .

Note: Logical operators such as || and && do boolean conversions internally, but actually return the value of original operands, even if they are not boolean.

Code examples: 

									
										2021 || 'JavaScript'	// implicit due to logical operator

										if ("value") { ... }      // implicit due to logical context

										let str = 'JavaScript' && 2021 // returns number 2021, instead of returning true

Double Negation

The double-negation !! is not a distinct JavaScript operator nor a special syntax but rather just a sequence of two negations. It is used to convert the value of any type to its appropriate true or false Boolean value depending on whether it is truthy or falsy.

Syntax: !!value

Note: many professionals consider the practice of using such syntax unacceptable and recommend simpler to read alternatives, even if they're longer to write.

Code examples: 

									
										!!1 // returns true

										!!0 // returns false

										!!{} // returns true

to Number conversion

The abstract operation ToNumber takes argument argument. It converts argument to a value of type Number according to Table 3:

Table 3: ToNumber Conversions

There are three different ways to convert into a number:

  • parseInt()
  • parseFloat()
  • Number()
  • Implicit conversion

parseInt ()

The parseInt () function parses a string and returns an integer. The radix parameter is used to specify which numeral system to be used.
If the radix parameter is omitted, JavaScript assumes the following:

  • If the string begins with "0x", the radix is 16 (hexadecimal)
  • If the string begins with "0", the radix is 8 (octal). This feature is deprecated
  • If the string begins with any other value, the radix is 10 (decimal)

Syntax: parseInt(string, radix)

Note:If the first character cannot be converted to a number, NaN is returned.

Code examples: 

									
										let str = '25px';

										let intValue = parseInt(str, 10);	// returns 25

										str = 'width: 25px';

										intValue = parseInt(str, 10);	// returns NaN

parseFloat ()

The parseFloat () function parses a string and returns a floating point number. This function determines if the first character in the specified string is a number. If it is, it parses the string until it reaches the end of the number, and returns the number as a number, not as a string.

Syntax: parseFloat(string)

Note: If the first character cannot be converted to a number, NaN is returned

Code examples: 

									
										let str = '2.55 rubles';

										let intValue = parseFloat(str);	// returns 2.55

										str = 'cost 2.55 rubles';

										intValue = parseFloat(str)	// returns NaN

Number ()

The Number () function converts the object argument to a number that represents the object's value.

Syntax: Number(object)

Note: Returns different object values to their numbers. If the value cannot be converted to a legal number, NaN is returned. If no argument is provided, it returns 0.

Code examples: 

									
										let str = '2.55 rubles';

										let intValue = Number(str);	// returns NaN

										str = '2021';

										intValue = Number(str)	// returns 2021

										let boolStr = true;

										intValue = Number(boolStr)	// returns 1

										boolStr = false;

										intValue = Number(boolStr)	// returns 0

Implicit conversion

Primitives are coerced to number when using

  • comparison operators 
    										
											"25" >= 15  -> 25 >= 15  -> true // numeric string with number
	
											"true" > 1  -> NaN > 1   -> false // text string with number
  • arithmetic operators (except for + when one operand is a string) 
    										
											"25" - 15  -> 25 - 15  -> 10 // numeric string with number

											undefined - 1 -> NaN - 1 -> NaN // undefined with number

											null + 2 -> 0 + 2 -> 2 // null with number
  • the unary + operator 
    										
											+'2021' -> 2021

											+true -> 1

											+undefined -> NaN

											+null -> 0
